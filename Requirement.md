이 문서는 차량 전자식 차일드락 개발을 위한 기능 초안입니다.
이 문서를 바탕으로, 소프트웨어 요구사항 명세서를 개발해야 합니다.

다음의 규칙을 준수합니다.
- 산출물은 SwRS2.md 입니다.
- A-SPICE, ISO26262 표준을 준수합니다.
- 차량용 용어를 사용하되, 약어인 경우 Full Name을 하단에 정리합니다.
- 가능한 자세하게 작성하며, 명확성, 일관성, 검증 가능성 등 요구사항의 일반 조건을 준수하세요.

기능 아이디어는 다음과 같습니다.
- 차일드락을 활성화하면 뒷 좌석을 차량 내에서 열 수 없습니다. 밖에서만 열 수 있습니다.
- 차일드락은 창문과 문을 함께 잠급니다.
- 차일드락은 왼쪽/오른쪽/양쪽을 구분하여 활성화 가능합니다.
- 차일드락은 화면에서 소프트 키로 활성화합니다.
- 후측방 센서로부터 신호를 받는 경우, 안전을 위해 자동 활성화합니다.
- 온도 센서를 이용해 화재로 식별 가능한 경우, 자동 비활성화 합니다.
- 내부 카메라를 이용해 후석에 어른이 앉은 경우, 자동 비활성화 합니다.
- 활성화는 음성 인식을 이용할 수 있습니다.
- 활성화 여부는 클러스터에 표시합니다.
- 이 외에 LLM은 추가 기능/비기능을 제안할 수 있으며, 이 때 LLM에 의한 제한이라고 표시해야 합니다.

SwRS.md에 포함할 항목은 다음과 같습니다.
- 전자식 차일드락 시스템 개요
- 이해관계자 식별
  - 이해관계자명, 식별 목적, 대상 팀(현대자동차 기준), 이해관계자 요구사항 추출 기법
- 이해관게자 요구사항 식별
- 기능 요구사항
  - Usecase 다이어그램 (Actor, Relation, Usecase, System 경계 표시 필수)
  - 각 Usecase에 대한 Usecase 명세서
  - 특히, Actor 식별 및 Actor에 대한 정의
  - 하나의 기능은 하나의 유스케이스로 작성
- 비기능 요구사항
  - ISO 25000 기준
  - 성능 요구사항의 경우, 최대 처리 시간 작성
  - 자원의 최대 사용률 작성
  - 상세한 검증 방법 포함
- 기능 안전 요구사항 (FMEA 기준)
  - ISO 26262 기준
